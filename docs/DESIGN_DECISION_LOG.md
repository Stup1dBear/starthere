# Design Decision Log

> Durable record of visual exploration rounds and what they taught the project.

## Purpose

This file stores the outputs of actual design exploration.

Use it to record:

- what was explored
- which directions were compared
- what was learned
- which direction is currently preferred
- what still needs validation

This file is not the final design style guide.

It is the evidence trail that should eventually inform one.

## When To Add An Entry

Add an entry when:

- a meaningful visual direction was explored
- a screen or flow was compared across multiple approaches
- an important design idea was rejected
- a design decision became stable enough to reference later

## Promotion Rule

Do not promote an entry directly into a permanent design rule just because it sounded good in discussion.

Promote only after:

- a visible artifact existed
- the comparison was real
- the preferred direction survived review

If a conclusion becomes stable, update the future design-style source of truth deliberately rather than assuming this log alone is enough.
